# Venky Dashboard — CMS Spec v2 (Venky-approved)

**URL:** https://venky-dashboard.netlify.app  
**Goal:** Single place to review, approve, and distribute all content (LinkedIn posts, carousels, blog articles).  
**Principle:** Approval here = action. No Slack approvals needed.

---

## Architecture

```
[Agents produce content] → [Convex DB] → [Dashboard reads/writes] → [Actions on approval]
     Eagle/Owl/Bee              ↕              Next.js app              ↓
     Stork/Crane            topics,         venky-dashboard        - LinkedIn: copy to clipboard
                            drafts,         .netlify.app           - Carousel: download PDF
                            blogs,                                 - Blog: auto-publish to AppletPod
                            feedback
```

**Stack:** Next.js 14 + Tailwind + Convex (already set up)  
**Deploy:** Netlify (site f3fa31de)  
**Data:** Convex (`tidy-crocodile-856.eu-west-1.convex.cloud`)

## Tabs (5 total)

### Tab 1: 📊 Dashboard (Overview)
**What:** Consolidated report. At-a-glance status of all pipelines.

| Section | Content |
|---|---|
| Stats row | Topics scouted (today/week), Drafts pending, Posts published (week), Blogs published (week) |
| Pipeline health | Last run time + status for each agent (Eagle, Owl, Bee, Stork, Crane, Pelican) |
| Action items | Items needing your review (count badge per type) |
| Recent activity | Last 10 actions across all pipelines |

### Tab 2: 📝 LinkedIn
**What:** Current pipeline tab. Scout → research → draft → approve → copy & post manually.

**Columns (Kanban — same as current static HTML pipeline):**
| Column | What's here | Actions |
|---|---|---|
| 🔍 Scouted | Eagle's topics | Approve (→ researched) · Archive · Reject with reason |
| 📋 Researched | Owl's briefs | Approve (→ drafting) · Reject |
| ✏️ Drafted | Bee's posts (250 words, 1 per topic) | Approve (→ ready) · Revise (send back to Bee) · Reject |
| ✅ Ready to Post | Approved posts | **📋 Copy Post** button · Mark Published |
| 🚀 Published | Posted content | Date posted |

**Card details (click to expand):**
- Full post text with markdown rendering
- Source link, engagement stats, fit score
- Connection tag (DIRECT/PASSIVE)
- Word count
- Copy button (copies clean text to clipboard)
- Feedback history

**LinkedIn is fully manual:** Copy post from here → paste into LinkedIn → Mark Published.

### Tab 3: 🎠 Carousels
**What:** Carousel pipeline. Similar approve/revise flow. PDFs stored in separate Git repo, downloadable from here.

**Columns:**
| Column | What's here | Actions |
|---|---|---|
| ✏️ Draft | Bee's carousel JSON | Preview slides · Revise · Reject |
| ✅ Approved | Ready carousels | **⬇️ Download PDF** · **📋 Copy Caption** · Mark Published |
| 🚀 Published | Posted carousels | Date posted |

**Card details:**
- Slide-by-slide preview (rendered from JSON)
- Caption text (separate from slides)
- PDF download button (file from Git repo)
- Caption copy button (for LinkedIn post text below carousel)

**Carousel PDF storage:** Separate Git repo (NOT AppletPod repo). PDFs generated by pipeline, committed to repo, download link in dashboard.

**Posting is manual:** Download PDF from here → upload to LinkedIn → paste caption → Mark Published.

### Tab 4: 📝 Blogs
**What:** Blog pipeline. Similar approve/revise states. On approve → blog team gets notified to publish on AppletPod.

**Columns:**
| Column | What's here | Actions |
|---|---|---|
| 🔍 Research | Stork's briefs | Approve for writing · Reject |
| ✏️ Draft | Crane's blog posts | Preview (rendered MDX) · Approve · Revise · Reject |
| ✅ Approved | Blog team notified to publish | View status |
| 🚀 Published | Live on appletpod.com/blog | View link |

**Card details:**
- Full blog preview (rendered markdown)
- Cluster tag (Interactive Design / AI+EdTech / Building Products)
- Keyword targets
- Word count
- SEO meta preview (title + description)

**Approval flow:**
1. You click "Approve" on a blog draft
2. Convex mutation → status = "approved"
3. **Notification sent to #blog-team** (C0APAUK3BEC) — Pelican picks it up
4. Pelican publishes to AppletPod site
5. Status updates to "published" with live URL

**No calendar needed for blogs (as of now).**

### Tab 5: 📅 Calendar
**What:** Content calendar for **LinkedIn + Carousels only**. No blogs.

**3-week view (Mon-Sat, Sunday off):**
- Each day shows: TEXT post or CAROUSEL (alternating)
- Color-coded: text = blue, carousel = purple
- Status indicators: ⏳ Scheduled · ✅ Ready · 🚀 Published · ⚠️ Gap
- Click to open the content card

**Schedule:**
- Mon/Wed/Fri → Carousel
- Tue/Thu/Sat → Text post
- Sunday → Off

## Global Features

### Authentication
- Simple password gate (existing — Venky enters password to unlock actions)
- Read-only without auth (can view but not approve/reject)

### Notifications
- On approve/reject → Slack notification to agent channel (already works)
- Badge counts on tabs for items needing review

### Data Sync
- Auto-refresh every 30 seconds (existing)
- Agents write to Convex → dashboard reads from Convex
- Dashboard writes approvals to Convex → agents read from Convex

### Mobile
- Responsive — works on phone (Venky reviews on mobile too)
- Touch-friendly approve/reject buttons
- Swipe-friendly card modals

## Decisions (Confirmed by Venky)

1. **Auth:** Simple password login. No Google/OAuth complexity.
2. **Blog approve:** Publish immediately. Blog team (#blog-team) gets notified → Pelican publishes to AppletPod.
3. **Carousel PDFs:** Stored in separate Git repo (NOT AppletPod repo). Download from dashboard.
4. **Old static HTML:** Delete AFTER new CMS is confirmed working.
5. **LinkedIn:** Fully manual. Copy from dashboard → paste on LinkedIn. No API.
6. **Calendar:** LinkedIn + Carousels only. No blog calendar.
7. **No separate carousel preview site needed.** Download PDFs from dashboard directly.
